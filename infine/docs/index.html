System requirements

If you're planning to build a website using Drupal, you must first ensure your hosting provider meets Drupal's requirements. Database
  • SQL 5.0.15 or higher with PDO
  • PostgreSQL 8.3 or higher with PDO
  • SQLite 3.3.7 or higher
PHP
  • PHP 5.2.5 or higher (5.3 recommended)
  • PHP5 GD library installed
  • Disabled error_reporting flag on production server
  • Enabled mod_rewrite in Apache for clean URLs
  • Memory limit set up at least 128MB

Installation Instructions

Using demo content

ZIP file which you downloaded from Themeforest is containing demo installation with all source codes and content including sample images. Note that the images are not same as on the preview server because of image license. Copy the demo folder from the downloaded file into the webroot and rename the directory. In the demo directory you can find database dump named database.sql. Connect to your database server and create database and user. Don't forget to add correct permissions to new user to access the database. After that you need to modify sites/default/settings.php file. You have to set correct database credentials. Locate the $database array and set proper credentials. Now you can import the dump file. For import you can use SequelPro or PhpMyAdmin. Below you can see working example from settings.php.


    $databases = array (
    'default' =>
    array (
    'default' =>
    array (
    'database' => 'database_name',
    'username' => 'database_user',
    'password' => 'database_password',
    'host' => '127.0.0.1',
    'port' => '',
    'driver' => 'mysql',
    'prefix' => '',
    ),
    ),
    );

With this type of installation you need realize that you are installing everything from our preview server (instead of images) so don't forget to set proper values for settings as email address, admin password, etc. Use this installation if you want exact copy of our preview installation. 

The login and password is admin and demo.

Content Management

Content slide

Content slide is simple piece of content presented in form of a slider. It holds image, title and some description to event. Content slides are accumulated by "Content slider" view and resulting block is named "View: Content slider".

Name Type Description
Title Textfield Title of content slide. This value is displayed trough "Content slider" view.
Title Subtitle Subtitle of content slide. This value is displayed trough "Content slider" view.
Body Textarea Body of content slide. This value is displayed trough "Content slider" view.
Image Image Picture representing slide picture. This value is displayed trough "Content slider" view.
Signature Textfield Signature for testimonial (example: "Social architect, Eduard Jones"). This value is displayed trough "Testimonials" view.

Sponsor

Simple content type representing sponsors trough linked image. Sponsors are displayed trough "Sponsors" view which creates Block with equal name.

Name Type Description

Link Link Link to sponsor site. This field wraps content of "Image" field. This value is displayed trough "Sponsors" view.

Gallery

Gallery is bounded with place. Each gallery of images should represent a gallery of "place", which is defined with taxonomy term. Actual display of galleries is done trough view "Gallery". Resulting block is called "Gallery".

Name Type Description
Title Textfield Title of gallery. This value is not displayed on the site.
Gallery Image Represents gallery images which are displayed trough colorbox integration. This value is displayed trough "Gallery" view.
Place Taxonomy term reference Places are defined in taxonomy vocabulary 'Place'. This value is displayed trough "Gallery" view.

  • Gallery view

Map

Map basically equals to 'marker' displayed on a map. Actual map is created by view "Map". Each new content of type "Map" creates new marker, according to latitude and longitude. Resulting block is called "View: Map".

Name Type Description
Title textfield Title of map. This value is not displayed on the site.
Longitude textfield Longitude of map location (Can be claimed from google maps). This value is important to display marker trough "Map" view;
Latitude textfield Latitude of company location (Can be claimed from google maps). This value is important to display marker trough "Map" view;

  • Map view

Sliders management

Revolution slider is provided by 'aviators_revolution' module and it provides flexible solution for creating and customizing sliders. Revolution slider creates 3 basic elements:

  • Slider: Systems allows to create infinite number of sliders
  • Slide: To each slider you can assign number of slides, which are consisting mostly of background image and components
  • Component: Component represents independent element on a slide. It might be caption, image, clock...

Creating a slider

Go on admin/revolution/slider ( Link is visible in main administration menu ). You will be able to see table of all sliders created in a system. New slider can be created by simply clicking on "Add Revolution slider" action link. Form with lots of configuration options will be displayed. All the form fields are well described and easy to understand. For most use-cases even the default values will be sufficient.

Creating a slide

Once slider is created, it will appear in the overview table and we can view it. Clicking on 'View' link will take us to the next screen. This screen will create preview of slider (If slides are available) and also displays table of slides created for this slider. You can change order of slides easily, just by using drag&drop handler in the overview table. To create a new slide, simply click on "Add new slide" action link. You will be taken to the slide form, which consists just of few fields. Probably most important field is 'image' field. This image will be used as a background of each slide.

Managing components

Clicking on 'View' link in overview table brings us to the detail of slide. This detail consist of 3 sections:

  • Add component action buttons Aviators revolution provides 3 components type by default
    • Caption - simple text
    • Image - simple image
    • Timer - countdown timer
    Each component type contains different field and different display. These component types can be created or updated at /admin/structure/revolution. The principle is same as creating/updating Node types.
  • Slide preview
    • The slide preview is created including all the components, so user can directly see what does this particular slide look like in action. Positioning of components can be simply done by dragging components around.
  • Component overview Table of all components.

Onepage management

For handling onepage-like websites in Drupal we created Infine Onepage module.

We wanted to provide end-user with possibilities like:

  • Quickly adding new "regions" and changing order of this regions.
  • Adding new rows, changing order of rows
  • Pushing content into rows, and structuring this content according to bootstrap framework.
  • Changing order of content in rows.
  • Create navigation using only anchor tags.

Infine onepage basically allows to create onepage layouts according to Boostrap scaffolding trough UI.

Multiple onepages can be created. However for this functionality is not fully developed and tested. For purposes of infine is single onepage perfectly suitable. Administration to onepage can be found at admin/structure/onepage. Clicking on manage for specific onepage takes us to layout of onepage. Layout contains Containers which are including Rows, which are including Blocks.

Containers can be sorted by simple Drag&Drop. This container is specified by bootstrap framework container definition. If you want for container to appear as a choice for menu item you need to fill in 'Wrapper ID' element when creating new container. Afterwards, when creating a new link in menu, new option will appear in section ONEPAGE NAVIGATION. You will be able to select your container, and onepage navigation link will be created.

Rows can be sorted by simple Drag&Drop. This row is specified by bootstrap framework row definition.

Block can be sorted by simple Drag&Drop. This component is specified by bootstrap framework spans definition.

To read more about Boostrap scaffolding, please follow this link: Boostrap scaffolding
